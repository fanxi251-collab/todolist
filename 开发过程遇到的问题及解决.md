## 1. 天气查询服务失败

### 问题描述

- 前几天查询天气成功，但今天查询失败
- 错误：`ConnectionResetError: 远程主机强迫关闭了一个现有的连接`

### 问题原因

网络连接不稳定，高德API请求被远程服务器强制关闭连接。

### 解决方案

1. 使用 `requests.Session()` 创建持久会话
2. 添加重试机制 (`urllib3.util.retry.Retry`)
3. 增加超时设置 (`timeout=10`)
4. 实现手动重试逻辑 (最多重试3次)

### 修改文件

`services/weather_service.py`

### 关键代码

```python
session = requests.Session()
retry_strategy = Retry(
    total=3,
    backoff_factor=1,
    status_forcelist=[429, 500, 502, 503, 504],
)
adapter = HTTPAdapter(max_retries=retry_strategy)
session.mount("https://", adapter)

# 请求时使用 session 替代直接使用 requests
response = session.get(AMAP_WEATHER_URL, params=params, timeout=10)
```

### 预防措施

- 添加更完善的错误处理
- 增加请求超时设置
- 实现指数退避重试

## 2. 数据库缺少新列

### 问题描述

- 添加新功能（附件、置顶）后启动失败
- 错误：`sqlite3.OperationalError: no such column: notes.attachment_path`

### 问题原因

项目已有旧数据库，新添加的字段未同步到数据库表结构。

### 解决方案

手动使用 `ALTER TABLE` 添加新列：

```sql
ALTER TABLE notes ADD COLUMN attachment_path VARCHAR;
ALTER TABLE notes ADD COLUMN is_pinned BOOLEAN DEFAULT 0;
```

### 修改文件

直接操作数据库文件 `todolist.db`

## 3. 删除标签失败

### 问题描述

- 删除标签时返回 404 "标签不存在"
- 标签实际存在

### 问题原因

删除标签时需要先解除与便签的关联关系（多对多关系），直接删除会因外键约束失败。

### 解决方案

在删除标签前，先清空关联关系：

```python
def delete_tag(db: Session, tag_id: int):
    db_tag = get_tag_by_id(db, tag_id)
    if not db_tag:
        return None
    
    # 先解除与便签的关联
    db_tag.notes = []
    db.commit()
    
    db.delete(db_tag)
    db.commit()
    return db_tag
```

## 4. 添加标签时自动创建便签

### 问题描述

- 点击"管理标签"按钮创建新标签后，莫名其妙创建了一个空便签

### 问题原因

- "管理标签"按钮和"+ 新建便签"功能混在一起
- 点击"管理标签"按钮时误触发了便签表单提交

### 解决方案

1. 分离便签和标签功能
2. 移除"管理标签"功能，保留"+ 新标签"按钮
3. 最终完全移除标签功能

### 修改文件

- `static/index.html` - 移除标签相关HTML
- `static/js/app.js` - 移除标签相关JS

## 5. Streamlit 警告

### 问题描述

- 启动时出现警告：
  ```
  Thread 'MainThread': missing ScriptRunContext! This warning can be ignored when running in bare mode.
  Session state does not function when running a script without `streamlit run`
  ```

### 问题原因

conda环境中安装了 streamlit 包，与项目无关但会输出警告。

### 解决方案

添加环境变量和警告过滤：

```python
import os
import warnings
warnings.filterwarnings("ignore")
os.environ["STREAMLIT_SERVER_HEADLESS"] = "1"
```

### 修改文件

`main.py`

## 6. 高德天气API返回空数据

### 问题描述

- 使用城市编码查询天气时返回空数据
- 错误：`casts: []`

### 问题原因

- 高德API使用城市编码（如101010100）查询预报时返回空数据
- 需要使用城市名称（如"北京"）

### 解决方案

修改请求参数，直接使用城市名称：

```python
params = {
    "key": api_key,
    "city": city,  # 使用城市名称而非编码
    "extensions": extensions,
    "output": "json"
}
```

`services/weather_service.py`
